# Flask-Based Fluff Generator

The fluff generator has been converted from Streamlit to Flask and now runs inline on the map page.

## ✨ What Changed

### **No More Separate Streamlit App**
- ❌ Removed dependency on Streamlit
- ✅ Fully integrated into Flask backend
- ✅ Runs directly on the map page

### **Inline Fluff Generator Modal**
- Opens on the map page without navigation
- Beautiful orange-themed interface matching the app design
- Real-time chat interface with message history

### **Updated Files**

**Modified:**
- `app.py` - Added Flask API endpoints for fluff generation
- `templates/red_scar_overview.html` - Added fluff generator modal and JS
- `templates/scars_watch.html` - Simplified (page still exists but redirects)

## 🚀 How It Works

### User Flow
1. User opens map at `/`
2. Clicks **⚡ Launch Fluff Generator** button
3. Modal opens on the same page
4. User types messages or battle reports
5. Responses generated by Claude API
6. Chat history maintained in session

### API Endpoints

#### `POST /api/fluff/message`
Sends a message to the fluff generator and gets a response.

**Request:**
```json
{
  "message": "What is the current state of Phlegmus?",
  "messages": [/* chat history */]
}
```

**Response:**
```json
{
  "response": "Generated response text...",
  "messages": [/* updated chat history */]
}
```

#### `POST /api/fluff/reset`
Resets the conversation history.

Create a `.env` file with your API key:

## 🐛 Troubleshooting

### "Error generating response"
- **Cause**: API key missing or invalid
- **Solution**: Check `.env` file has valid `GEMINI_API_KEY`

### "No ongoing story yet"
- **Cause**: `data/ongoing_story.txt` not found
- **Solution**: Make sure the file exists in the project

### Modal doesn't open
- **Cause**: JavaScript error
- **Solution**: Check browser console (F12 → Console tab)

### Slow responses
- **Cause**: First API call takes time
- **Solution**: Subsequent calls are faster due to caching

## 🎯 Integration with Backend

The fluff generator integrates with:
- `src/model_funcs.py` - Initializes Gemini model
- `src/prompts.py` - System and initial prompts
- `src/retrieval.py` - Loads ongoing story context
- `data/ongoing_story.txt` - Story context for generation

### How It Works
1. User sends message
2. Flask API retrieves story context
3. Message sent to Gemini with system prompt
4. Response generated in Warhammer 40k voice
5. Response displayed in modal

## 🚀 Performance

- **Response Time**: 2-5 seconds typical (depends on Gemini API)
- **Chat History**: Stored in browser (max reasonable size ~10K tokens)
- **No Page Reload**: Everything happens in modal
